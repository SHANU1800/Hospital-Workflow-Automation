## MCP design rules before adding 100+ tools

- Keep tool names **verb_noun** (`assign_bed`, `create_claim`, `send_notification`).
- Every tool should return a standard envelope:
  - `success`, `error`, `data`, `meta` (trace_id, latency_ms, source)
- Add idempotency where needed (`idempotency_key`) for retries.
- Tag tools by domain (`clinical`, `ops`, `finance`, `compliance`, `integration`).
- Keep agents thin; tools do heavy system interaction.
